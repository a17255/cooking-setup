# Chatbot Design — Cooking Setup

**Date:** 2026-04-26  
**Status:** Approved

---

## Overview

Add a floating chatbot to the GitHub Pages cooking site (`docs/index.html`) that lets users query and interact with their meal list using natural language (Vietnamese and English). Powered by Google Gemini (gemini-2.0-flash), called directly from browser JavaScript.

---

## Decisions

| Question | Decision |
|---|---|
| Deployment | Static GitHub Pages — no backend |
| LLM | Google Gemini API (gemini-2.0-flash), free tier (1,500 req/day) |
| Language | Auto-detect — reply in same language user writes in |
| UI | Floating 💬 button, bottom-right, expands to panel |
| Architecture | Separate `docs/chatbot.js` (never overwritten by Python generator) |

---

## Architecture

```
docs/
  index.html        ← generated by html_report.py; has <script src="chatbot.js">
  chatbot.js        ← NEW: all chatbot logic; static file, never regenerated

html_report.py      ← modified: inject <script src="chatbot.js"> into template
.gitignore          ← add .superpowers/
```

**Data flow:**

```
User types message
  → chatbot.js reads meal arrays from page JS variables
  → fetch() to Gemini API with system prompt (meal list + today's menu + instructions)
  → Gemini returns JSON: { action, response, data }
  → chatbot.js displays response text + executes action on page
```

**Why Gemini handles all routing:** Simpler than a local intent router; handles Vietnamese typos/slang naturally.

---

## Chat UI

- **Collapsed:** Gold 💬 button fixed `bottom: 24px; right: 24px`. Red badge for first-open hint.
- **Expanded:** Panel anchored bottom-right (`width: 340px; height: 480px`). Panel does not cover full screen.
- **Header:** "🍳 Trợ lý nấu ăn" + ⚙ settings icon + ✕ close button.
- **Colors:** Match existing page theme — dark background (`#23233a`), gold accent (`#e8c97e`).
- **First message on open:** Greeting + usage hint in Vietnamese.

## Features (all 8)

| Feature | Example input | Gemini action | chatbot.js effect |
|---|---|---|---|
| 🔍 Keyword search | "sưởn", "thịt bò cà ri" | `chat` | Display matching dishes from list |
| 📊 Count & stats | "có bao nhiêu món chính?" | `chat` | Count from meal data, list by category |

## Gemini Integration

**Model:** `gemini-2.0-flash`  
**Endpoint:** `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=API_KEY`  
**Call type:** Direct `fetch()` from browser JS

**System prompt structure:**
```
You are a Vietnamese family meal planning assistant embedded in a cooking website.

Today's menu: {main} | {side} | {soup}

Available meals:
- Main dishes ({n}): {comma-separated list}
- Side dishes ({n}): {comma-separated list}
- Soups ({n}): {comma-separated list}

Always respond with valid JSON only:
{
  "action": "chat" | "reroll_main" | "reroll_side" | "reroll_soup" | "add_dish",
  "response": "your reply to the user",
  "data": { "category": "main|side|soup", "name": "dish name" }
}

Rules:
- Reply in the same language the user writes in (Vietnamese or English)
- For reroll actions, still include a friendly response explaining the change
- For add_dish, confirm what you're adding and which category
- data is only required for add_dish action; omit or null for others
```

## API Key Flow

1. User opens chat for the first time → panel shows setup screen (not a modal overlay)
2. Setup screen: text input for key + link to `aistudio.google.com` to get free key
3. Key saved to `localStorage['cooking_gemini_key']`
4. ⚙ icon in header re-opens setup screen to update or clear the key
5. Key is never sent anywhere except the Gemini API endpoint

---

## Error Handling

| Scenario | Behavior |
|---|---|
| No API key set | Show setup screen instead of sending |
| Gemini API error (4xx/5xx) | Show: "Chatbot tạm thời không khả dụng 😅 Thử lại sau nhé." |
| Quota exceeded (429) | Show: "Đã dùng hết quota hôm nay. Thử lại ngày mai nhé." |
| Invalid JSON from Gemini | Fall back to displaying raw `response` text |
| Network offline | Catch fetch error, show offline message |
| Unknown action returned | Treat as `chat`, display response only |
